我想要创建一个 PaperShape 新组件，或许可以结合一个 playground 或者是 code generator，去生成不同的边框角落的一些形状的组件。比如：邮票，优惠券，门票/小票，吊牌，书签，纸张，撕纸，折角，叠纸，剪纸边缘，拼贴块，缝线边缘。
形状的定制可以从 Corner，Edge(除了形状，还有包含线条粗细) , Roughness, Cutout (给某个特定区域进行裁剪，比如优惠券中间打孔，)
这个组件可以开个新的 example 页面展示，以及配上一个 playground 帮助用户构建想要的形状，可以参考下 app/ui/pattern-monster-presets 的实现思路

这种异形除了填充本身，可能还要支持边框、阴影等多个层级组合在一起。所以不仅要考虑单一图层，还要考虑：
1. 多个图层之间的特殊样式实现，每个图层可以单独控制，支持一些形变, 实现 hover,active 等变化
2. 响应式的支持，和动态内容的尺寸变化
3. 整体的一个性能，比如大量用 drop-shadow blur 会造成卡顿？想想性能更好的实现

总之我的诉求就是，因为我这个是手帐风，想要一些手工感。我不希望就是那种简单的圆角矩形，或者是一个简单的圆形，这样太过于扁平跟机械了。我增加一点手工的温度。

手账风相对于那种科技风，它的麻烦点在于手工感的构造会比较复杂。比如那个折角，它并不是直接把纸撕掉，而是需要把角落折一下，可控制折角大小。这里可能还要考虑到光影，尤其是折完之后光影的变化、斜度等，处理起来就会比较麻烦。

再比如，如果我想要某一边是波浪纹的：
1. 为了考虑到手感，那个波浪纹不能像 SVG 矢量图那种非常重复、平铺的 pattern。
2. 它可能还需要有一些真实手撕的那种感觉。
4. 撕掉的那一部分。如果还要再给它加边框或者加一些 outline 什么的，感觉会有点复杂。

另外一块的话，我想再新增一个 Paper Pattern (有别与 Pattern Monster) 的组件配合 Paper Shape 一起使用，作为类似页面的纸张背景。比如有横线纸，方格纸，点阵纸，乐谱纸，儿童涂鸦张，手帐纸。我在 Resume 项目中有简单实现一版你可以参考下 docs/paper-shape/paper-texture.tsx，实现更多手帐风背景，支持自定义 Spacing, Size, Stroke, Color 等参数

然后还有的话就是我希望在 Playground 支持结合各种 Overlay Layer 作为 Decoration，比如弄一写图钉，回形针，胶带，贴纸，缝线，有机/自然类（叶片，云边，波浪，石头），几何装饰

完整的架构应该是 PaperShape + PaperPattern + Decorations，能够在 Example 看场景的调用，在 Playground 中去自由组合出想要的效果，然后输出相应的代码，代码除了有对组件的调用版本，还可以输出独立的代码更方便后续定制，不侵入原有组件，现在很多 UI Kit 就是这么做的

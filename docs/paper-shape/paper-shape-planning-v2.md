# Paper Shape Planning V2（童趣日式可爱手绘风重构）

## 0. 背景与目标

本次 V2 目标：基于现有 `paper-shape` 体系，重做一版视觉与几何都达标的组件系统，风格明确为：

1. 童趣
2. 日式可爱
3. 手绘感
4. 可用于真实 UI 场景（不是只看图）

参考输入：

1. 失败样例（V1）：`docs/paper-shape/images/codex-ui-paper-shape-examples-2026-03-26-19_06_15.png`
2. 风格参考图目录：`docs/paper-shape/images/ui-refs`
3. 规范参考：`docs/craft-ui-spec.md`

---

## 1. V1 问题复盘（为什么要重做）

从当前 V1 结果看，核心问题不是“页面功能不全”，而是“形状和风格没有做对”：

1. 形状识别度不足：不同 preset 的轮廓差异太弱。
2. 关键特征错误：如邮票齿边、小票锯齿、优惠券打孔等不稳定或不明显。
3. 手绘感单一：更像统一模板换参数，而不是不同纸品语义。
4. 装饰和版面太“程序化”：缺少童趣、可爱、手工拼贴的氛围。
5. 缺少视觉验收基线：有功能但没有“像不像参考图”的硬标准。

结论：V2 必须按“视觉达标优先”实施，不再按纯工程模块推进。

---

## 2. V2 风格锚点（必须统一）

### 2.1 总体视觉方向

1. 线条：手绘笔刷感，不是机械矢量线。
2. 色彩：暖米色/奶油色纸底 + 柔和粉绿蓝点缀。
3. 纹理：低强度纸张颗粒 + 局部蜡笔/彩铅感。
4. 装饰：胶带、回形针、贴纸、孔眼、缝线等要“可爱化”。
5. 文字：参考 `craft-ui-spec` 的中文手写风组合（标题与正文区分）。

### 2.2 形态语法（参考图抽象）

第一批优先语义：

1. `stamp`（邮票齿边）
2. `coupon`（双侧打孔）
3. `ticket`（票根 + 切口）
4. `receipt`（底边锯齿）
5. `tag`（吊牌 + 孔眼 + 绳）
6. `folded`（折角）
7. `torn`（撕边）
8. `stitched`（缝线边）

第二批扩展语义：

1. `layered-paper`
2. `cutout-edge`
3. `collage-piece`
4. `bookmark`

## 3. 架构策略（重写优先，有限复用）

若 V1 代码影响风格落地，V2 采用“核心重写、外围复用”策略。

### 3.1 建议重写部分

1. 几何编译内核（edge/corner/cutout 规则）
2. preset 规则层（每个 preset 的专属几何约束）
3. 视觉 token 层（线条、纹理、阴影、色板）
4. 示例体系（示例库 + 详情页）

### 3.2 可复用部分（按质量判断）

1. 导出工具（Recipe/JSX/SVG）
2. 页面导航壳层与路由结构
3. Stack / Book 的基础组合结构

原则：复用不能降低最终风格质量。

### 3.3 外部参考使用原则（不内置摘录）

V2 保留参考来源，但不在本文件内置大段摘录；执行时按来源文件实现：

1. `docs/paper-shape/augmented-ui.css`：参考变量化拼装与分层思路。
2. `docs/paper-shape/com-refs/experience-timeline.tsx`：参考边缘裁剪与 pattern 叠层。
3. `docs/craft-ui-spec.md`：参考整体风格、字体与材质方向。

实现约束（仍然必须满足）：

1. `fill / stroke` 独立分层。
2. 四角四边保持独立参数结构。
3. 几何计算与图层渲染分离，支持同几何多层复用。

## 5. Preset 规范（V2 首批）

每个 preset 都要定义：

1. 核心特征
2. 角规则
3. 边规则
4. cutout 规则
5. humanize 上限
6. 禁止项

### 5.1 首批规范表（按参考图细化）

说明：首批按你给的 4 张参考图收敛，优先保证“识别度 + 可爱度 + 手绘感”。

| preset | 核心特征（必须命中） | 几何参数建议 | 可选装饰 | 禁止项 |
| --- | --- | --- | --- | --- |
| stamp | 四边连续邮票齿边，整体小矩形比例 | `toothCount`: 10~18（随边长）；`toothDepth`: 边长 2.8%~4.2%；四角 `round(4~10)` | 小印章图标、轻线稿、边角小花 | 禁止出现平直边占主导；禁止齿深过浅导致“普通卡片感” |
| coupon | 左右半圆打孔 + 顶部中间小缺口（参考图更明显） | 左右孔半径：短边 7%~10%；上中缺口半径：短边 4%~6%；边框圆角 8~14 | 虚线撕线、轻图标、折扣字贴纸 | 禁止仅做左右孔不做中间缺口；禁止孔位偏离中轴 |
| ticket | 票根切口语义：左右内凹 + 一侧轻撕边或齿边 | 左右切口半径：短边 8%~11%；右边缘可选 `tear(amp 3~7)`；上边可微 notch | 条码、编号、轻虚线分割 | 禁止做成 coupon 同款双孔；禁止过度圆角化 |
| tag | 吊牌轮廓 + 单孔 + 挂绳空间 | 默认 6/8 边切角轮廓；孔径：短边 6%~9%；孔位偏上角安全区 | 绳结、按钮、贴纸、小图标 | 禁止做成长方形无孔；禁止孔位贴边 |
| folded | 单角折角显著，折角有双层阴影与高光 | 折角边长：短边 16%~24%；折角内层偏移 1~3px；主体角可 0~8 | 胶带、轻线条、便签字 | 禁止折角过小不可见；禁止折角穿出主体 |
| torn | 至少一条长边明显撕裂，不规则波峰波谷 | `tearAmp`: 4~10；`tearFreq`: 中低频；顶/底边至少一边 tear | 胶带固定、图钉、纸纤维纹理 | 禁止规则正弦波过于机械；禁止四边全 tear 导致脏乱 |
| stitched | 主体平稳 + 内圈虚线/缝线边框 | 外轮廓圆角 8~14；内圈 inset 6~12；缝线间距 8~14 | 小花/按钮/角标 | 禁止只有外框没有内圈缝线；禁止线条太硬太黑 |
| scalloped-edge | 扇贝花边边框（参考图重点） | scallop 半径 8~16；间距=半径*1.8~2.2；可单边或四边 | 小花、点缀符号 | 禁止 scallop 过密形成噪点；禁止与 stamp 齿边混淆 |

### 5.2 业务扩展项（不挡首批）

| preset | 说明 |
| --- | --- |
| receipt | 业务常用：顶部平直 + 底边锯齿，可作为首批后半段并行实现 |
| basic-paper | 基础纸张（轻切角/轻卷角）用于承载 layered/collage 组合 |

## 6. 视觉 Token 体系（V2 强约束）

### 6.1 颜色与材质

1. 主基调改为“奶油中性色 + 糖果浅彩”，不是纯暖黄主导。
2. `paper.base.*`：奶油白、云朵白、浅粉米、浅杏灰（保留少量浅牛皮纸作为辅色）。
3. `ink.stroke.*`：柔和棕灰/可可色线稿（避免纯黑硬线）。
4. `accent.sweet.*`：蜜桃粉、薄荷绿、天空蓝、薰衣草紫、奶橘色。
5. 推荐配色占比：`base 60% / sweet accent 30% / contrast 10%`。
6. 禁止项：大面积单一暖黄导致“旧纸发暗感”，与童趣甜系目标冲突。
7. `texture.paper.*`：细颗粒、轻纤维、轻污点（低对比）。

### 6.2 线条与阴影

1. 主轮廓线宽区间：`1.4 ~ 2.4`（可局部波动）。
2. 线条不均匀度：可控但不破坏轮廓识别。
3. 阴影策略：短偏移软阴影，不使用重 blur。

### 6.3 装饰与排版

1. 装饰安全区：避免压住关键轮廓特征（孔、齿、折角）。
2. 标签文字：限制最大行数与最小字号，确保可读。
3. 图标风格：可爱化、圆润化、低攻击性对比。

---

## 7. Humanize 与 Random 策略（V2）

### 7.1 Humanize（结构优先）

1. 仅在允许区域扰动。
2. 扰动后必须通过几何约束检查。
3. `seed` 保证可复现。

### 7.2 Random（编辑器核心能力）

1. 提供“骰子按钮”随机生成参数。
2. 随机必须服从 preset 规则，不得破坏语义。
3. 提供可选锁定：`锁 preset`、`锁尺寸`、`锁装饰`（V2.1 可加）。

## 8. 实施阶段（按可验收结果推进）

### 阶段 A：视觉与几何基线（必须先完成）

1. 先做 `stamp/coupon/ticket/tag` 四个 preset（参考图第一行语义最强）。
2. 完成每个 preset 的目标图对照 + 禁止项校验。
3. 未通过视觉验收，不进入阶段 B。

### 阶段 B：核心组件与示例体系

1. 重建 `PaperShape` 渲染管线（轮廓、纹理、装饰）。
2. 补齐 `folded/torn/stitched/scalloped-edge` 四个 preset。
3. 导出一致性检查（页面预览与 SVG 一致）。

### 阶段 C：参数编辑器与组合

1. 重做 Playground（中文命名“参数编辑器”）。
2. 加入 Random、图层开关、装饰控制、参数分组。
3. 完成 `Stack` 页面的纸堆与书册视觉升级，并并行补 `receipt/basic-paper`。

### 阶段 D：性能与回归

1. 关键 preset 快照回归。
2. benchmark 与缓存策略。
3. 导出质量与跨尺寸一致性。

## 9. 验收标准（V2 必须可判断）

### 9.1 视觉验收

1. 首批 4 个 preset 肉眼可快速识别。
2. 与目标图比对时，核心特征一致。
3. UI 整体呈现“童趣日式可爱手绘风”，而非通用卡片风。

### 9.2 几何验收

1. 关键结构（齿边/打孔/锯齿/折角）稳定。
2. 不同尺寸不破型。
3. seed 可复现。

### 9.3 工程验收

1. 示例列表支持快速复制与导出。
2. 示例详情支持微调与导出。
3. 参数编辑器支持 Random 随机生成。
4. 性能测试页面可读、有可对比指标。
